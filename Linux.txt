# PART 5 — Linux Permissions (MOST IMPORTANT SECTION)

This is one of the highest probability topics.

---

# Permission Structure

Example:

```bash
-rwxr-xr--
```

Breakdown:

```bash
- rwx r-x r--
```

| Section | Meaning |
| ------- | ------- |
| owner   | rwx     |
| group   | r-x     |
| public  | r--     |

---

# Permission Meanings

| Symbol | Meaning |
| ------ | ------- |
| r      | read    |
| w      | write   |
| x      | execute |

---

# Numeric Permission System

| Permission | Value |
| ---------- | ----- |
| r          | 4     |
| w          | 2     |
| x          | 1     |

---

# Example

## rwx

```text
4 + 2 + 1 = 7
```

## r-x

```text
4 + 1 = 5
```

## r--

```text
4
```

So:

```bash
rwxr-xr--
```

becomes:

```bash
754
```

---

# chmod Command

Change permissions.

Syntax:

```bash
chmod permission filename
```

---

# MOST IMPORTANT chmods

## 777

```bash
chmod 777 file.txt
```

Meaning:

* everyone full access

---

## 755

```bash
chmod 755 script.sh
```

Meaning:

* owner full
* others read + execute

VERY common.

---

## 644

```bash
chmod 644 file.txt
```

Meaning:

* owner read/write
* others read only

VERY common.

---

# Symbolic chmod

Instead of numbers.

---

# Add execute permission to owner

```bash
chmod u+x script.sh
```

---

# Remove write from group

```bash
chmod g-w file.txt
```

---

# Add read to others

```bash
chmod o+r file.txt
```

---

# PART 8 — MOST IMPORTANT EXAM PATTERN

This EXACT style appeared in your sample paper.

---

# Example Problem

Create:

* doc1.txt to doc15.txt

Apply special permissions to files divisible by 3.

---

# FULL SOLUTION

```bash
for i in {1..15}
do
  touch doc$i.txt

  if (( i % 3 == 0 ))
  then
    chmod 764 doc$i.txt
  else
    chmod 640 doc$i.txt
  fi
done
```

---

# Understand This Carefully

## if condition

```bash
if (( i % 3 == 0 ))
```

Checks:

> divisible by 3

---

# then

Runs commands if condition true.

---

# else

Runs otherwise.

---

# fi

Ends if block.

---

# THIS TYPE IS VERY HIGH PROBABILITY

They may change:

* divisible by 2
* divisible by 5
* even/odd
* naming pattern
* permission values

But logic stays SAME.

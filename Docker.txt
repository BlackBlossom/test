# PART 1 — What is Docker?

## Problem Before Docker

Different systems behave differently:

```text id="p1dock"
Works on my machine ❌
Fails on server ❌
```

---

# Solution

Docker packages:

* app
* dependencies
* runtime
* environment

into a:

# Container

So app runs SAME everywhere.

# PART 6 — Port Mapping (VERY IMPORTANT)

HIGH probability practical.

---

# Problem

Container runs internally.

Need external access.

---

# Solution

Port mapping.

---

# Syntax

```bash id="d13dock"
docker run -p HOST:CONTAINER image
```

---

# Example

```bash id="d14dock"
docker run -d -p 8080:80 nginx
```

Meaning:

| Part | Meaning        |
| ---- | -------------- |
| 8080 | host port      |
| 80   | container port |

# PART 8 — Dockerfile (EXTREMELY IMPORTANT)

THIS is probably the MOST important Docker topic.

Very high chance in practicals.

---

# What is Dockerfile?

A text file containing instructions to build Docker image.

---

# Basic Dockerfile

```dockerfile id="d17dock"
FROM node:18

WORKDIR /app

COPY . .

RUN npm install

CMD ["npm", "start"]
```

---

# Understand Every Line

---

# FROM

```dockerfile id="d18dock"
FROM node:18
```

Base image.

---

# WORKDIR

```dockerfile id="d19dock"
WORKDIR /app
```

Working directory inside container.

---

# COPY

```dockerfile id="d20dock"
COPY . .
```

Copies project files.

---

# RUN

```dockerfile id="d21dock"
RUN npm install
```

Executes command during image build.

---

# CMD

```dockerfile id="d22dock"
CMD ["npm", "start"]
```

Runs when container starts.

VERY IMPORTANT difference from RUN.

---

# RUN vs CMD

| RUN        | CMD               |
| ---------- | ----------------- |
| build time | container runtime |

HIGH probability MCQ.

---
